# OncoTwin-GBM

Computational oncology digital twin bridging wet-lab CDC25 phosphatase inhibitor
assays (crystal violet, U87/U251) with docking-based virtual screening for
glioblastoma drug discovery.

## Status: scaffold running on SYNTHETIC placeholder data

Every number in `data/*.csv` right now is fake, generated by
`data/generate_placeholder_data.py`, purely so the pipeline can be built and
verified end-to-end. **Nothing here should be cited or presented as real
until real data replaces it.**

### To plug in real data
1. Export your crystal violet results and AutoDock Vina docking results as CSV
   (or point `src/data_loader.py` at your `.xlsx` files directly).
2. Check the column names against `EXPECTED_CV_COLUMNS` /
   `EXPECTED_DOCKING_COLUMNS` in `src/data_loader.py`. If they differ, either
   rename your columns or fill in `CV_RENAME_MAP` / `DOCKING_RENAME_MAP`.
3. Replace `data/crystal_violet_raw.csv` and `data/docking_results_raw.csv`
   with your real files (same filenames, or update the paths).
4. Re-run the pipeline (see below) — everything downstream (IC50 fits,
   screening model, dashboard) updates automatically.

## Architecture

This fixes a specific flaw in early drafts of this project: a single model
cannot be trained on wet-lab assay features and then queried with completely
different docking/molecular features. Instead, there are two connected
stages:

```
crystal_violet_raw.csv                    docking_results_raw.csv
        |                                          |
        v                                          v
  wetlab_ic50.py                          screening_model.py
  (4PL dose-response curve fit,          (aggregate docking scores +
   NOT machine learning -- too few        molecular descriptors per
   points per curve for ML)               compound)
        |                                          |
        +-------------------> anchor <-------------+
                          (compounds with
                        BOTH measured IC50
                         AND docking data)
                                |
                                v
                     train RF/GBM: features -> IC50
                                |
                                v
                  predict IC50 for virtual-only
                     compounds (no wet-lab data)
                                |
                                v
                  explainability.py (SHAP on the
                     screening model, not the
                     wet-lab curves, which are
                     already interpretable)
                                |
                                v
                        app/app.py (Streamlit)
```

### Why the wet-lab step isn't ML
Crystal violet gives ~6-8 concentration points per compound/cell line,
replicated 2-3x. That's a classic pharmacology dose-response curve-fitting
problem (4-parameter logistic), not enough independent samples for a
regression model. Random Forest/Gradient Boosting are used only at the
virtual screening stage, where the samples are *different compounds* and the
features are their molecular/docking descriptors.

### The anchor-set limitation (important for the thesis writeup)
The screening model can only be trained on compounds where you have *both*
measured IC50 (from crystal violet) and docking data — the "anchor set."
With a small number of anchor compounds (e.g. just NSC95397 initially), the
model has very limited statistical power. The code reports leave-one-out
cross-validated R² when possible and prints an explicit warning below n=10.
**Do not present this as a validated predictive model until the anchor set
is large enough to support real cross-validation** (ideally 15-20+ compounds
with measured IC50). Until then, describe it as a preliminary prioritization
ranking, and be explicit about that scope in any write-up or slide.

## Running the pipeline

```bash
pip install -r requirements.txt

# 1. (optional) regenerate synthetic placeholder data
python data/generate_placeholder_data.py

# 2. run each stage standalone to inspect outputs
cd src
python wetlab_ic50.py          # -> data/wetlab_ic50_table.csv
python screening_model.py      # -> data/virtual_screening_ranked.csv, anchor_dataset.csv
python explainability.py       # -> data/shap_summary.png

# 3. launch the dashboard
cd ../app
streamlit run app.py
```
